🔍 DevTools Debugger — Manual Verification

Open Chrome DevTools → Sources panel. Ensure ?scheme-debug=true is in the URL or a <script> tag below has the debug attribute.

Test Sources

Execution results will appear in the DevTools console.

Verification Checklist

Work through each scenario. Check the box when verified.

Core Debugging

Steps

Set a breakpoint on the (if (<= n 1) ...) line in factorial. Reload → verify execution pauses at that line.

Steps

Pause at (compute 4). Step Into → should enter the body of compute. Step Into again → should enter (* x x) or (factorial x).

Steps

Pause inside compute. Step Over (factorial x) → it should evaluate without entering factorial's body, stopping at (sum-tail x 0).

Steps

Pause at (* n (factorial (- n 1))). Step Into repeatedly → should progress through sub-expressions: (- n 1), (factorial ...), etc.

Steps

Pause inside factorial. Click Step Out → should resume and pause at the caller's next expression (e.g., back in compute).

Source Types

Steps

Look in Sources for scheme://inline-scripts/script-0.scm. Should show the inline Scheme code with correct line numbers.

Steps

Look in Sources for scheme://scheme-sources/manual_script.scm. Should show the external Scheme file content.

Steps

Set breakpoints in both inline and external scripts. Verify both fire correctly on reload.

Variable Inspection

Steps

Pause inside factorial with n=5. Check the Scope pane → should show n: 5 under "With Block".

Error Handling

Steps

Add (error "test error") to a script. Enable "Pause on caught exceptions" in DevTools. Verify DevTools pauses at the Scheme source location, not internal JS code.

Tail Call Optimization

Steps

Set a breakpoint inside sum-tail. Let it hit a few times → verify no stack overflow occurs. The extension sidebar should show TCO badges.

Async Stack / Extension (Requires Extension)

Steps

Pause inside a nested call (e.g., factorial called from compute). Check the native Call Stack → should show "Async" separator with scheme: factorial, scheme: compute labels.

Steps

Pause in nested call → the Scheme Stack sidebar should show the full call stack with function names and source locations.

Steps

Click a non-top frame in the sidebar → the editor should navigate to that frame's source file and line.

Steps

Click a non-top frame → the Variables section should show that frame's local bindings, not the top frame's.

Steps

Pause in sum-tail → sidebar should show a TCO×N badge on the tail-called frame.

Steps

Use the Step Into/Over/Out buttons in the sidebar → verify correct Scheme-level stepping (not JS-level).

Console API

Steps

While paused, type __schemeDebug.getStack() in Console → should return array of frame objects with name, source, tcoCount.

Steps

While paused, type __schemeDebug.stepInto() then press Resume → should step into the next Scheme expression.

Steps

While paused inside factorial with n=5, type __schemeDebug.eval("(+ n 1)") → should return "6".
